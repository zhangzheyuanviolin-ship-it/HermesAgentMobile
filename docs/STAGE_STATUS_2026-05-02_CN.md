# HermesAgentMobile 阶段状态说明

## 当前结论

截至 2026-05-02，HermesAgentMobile 已完成第一阶段正式落地，当前 `v0.3.21` 可作为第一版正式发布基线。

本阶段最重要的成果有三件：

1. 原始正式版与并行测试版两条安装链路已经明确分离。
2. 聊天页、多模型、会话、附件和后台网关形成了可连续使用的闭环。
3. Shizuku 系统命令能力已完成接入，并解决了“界面显示已授权但后台命令不可用”的一致性问题。

## 本阶段关键修复结论

### 1. 正式版与测试版不再互相覆盖

两个包名已经彻底分离，测试版可用于高风险功能验证，正式版保持稳定。

### 2. Shizuku 状态不再是“假已授权”

当前设置页状态不再只看 Android 授权，而会同时核验：

- Shizuku 是否已安装
- 服务是否运行
- 权限是否授予
- 通道开关是否开启
- 容器内 `system-shell` 和 `system-shell-status` 是否真实存在并可执行

### 3. 网关残留与错误接管问题已修复

当前正式版已经补强：

- 停止网关时清理残留 proot / Python 进程
- 删除过期 `gateway.pid`
- 启动时先补环境、补桥接脚本，再决定是否接管已有实例
- 网关运行状态改为真实进程 / 真实端口校验，不再长时间虚报“运行中”

## 发布建议

当前建议以 `v0.3.21` 作为仓库第一版正式 Release。

原因：

- 原始正式版已完成实机验证
- Shizuku 系统命令能力已正常工作
- 设置页状态与后台实际状态已经对应
- 当前版本可作为以后继续迭代的稳定底座

## 后续继续开发时的建议

1. 正常新功能一律先走测试版分支与测试包名。
2. 仅在测试版实机验证通过后，再回灌到正式版分支。
3. 与聊天页、解析器、网关服务、Shizuku 状态相关的改动，必须联动检查“前端显示、原生状态、容器命令、后台进程”四个层面。
4. 后续若继续做大改版，优先保留这套双链路发布策略，不要再合并成单链路。
